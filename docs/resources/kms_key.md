# thalassa_kms_key (Resource)

Create and manage a KMS key in Thalassa Cloud

~> **Early access:** KMS is in early access. The API and Terraform resource schema may change in future provider releases.

See [KMS product documentation](https://docs.thalassa.cloud/docs/kms/).

KMS must be enabled for your organisation. Use the `thalassa_kms_summary` data source to verify regional availability before creating keys.

## Deletion behaviour

`DeleteKey` schedules deletion; the key enters `pending_deletion` status. Set `cancel_scheduled_deletion = true` and apply to cancel a pending deletion.

## Example Usage

```terraform
resource "thalassa_kms_key" "app" {
  region   = "nl-01"
  name     = "app-secrets"
  key_type = "aes256-gcm96"

  key_rotation_enabled    = true
  rotation_period_in_days = 90
}
```
